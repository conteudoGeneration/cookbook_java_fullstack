<h1>Box Model</h1>



No CSS, **todo elemento é tratado como uma caixa (Box)**. Independentemente de ser um título, parágrafo, imagem ou `div`, o navegador renderiza cada elemento seguindo o **Modelo de Caixa (Box Model)**. Esse modelo define como o elemento ocupa espaço na tela e como se relaciona visualmente com os demais elementos do layout.

O Box Model explica o **dimensionamento e o espaçamento** dos elementos por meio de um conjunto de propriedades fundamentais do CSS.

### Estrutura do Box Model (de dentro para fora)

- **Área de conteúdo (content)**
- **Padding (preenchimento interno)**
- **Border (borda)**
- **Margin (margem externa)**

Veja a representação visual abaixo:

<div align="center"><img src="https://i.imgur.com/qTgX5zY.png" title="source: imgur.com" /></div>

- **Conteúdo (content)**: É a área onde o conteúdo do elemento é exibido, como texto, imagens ou outros elementos HTML. O tamanho dessa área pode ser controlado por propriedades como `width` e `height`.
- **Padding**: Define o espaço **interno** entre o conteúdo e a borda do elemento. Ele aumenta a área visível do elemento, mas não afasta outros elementos ao redor.
- **Border**: É a borda que envolve o padding e o conteúdo. Pode ter largura, estilo e cor definidos. Ela marca visualmente o limite do elemento.
- **Margin**: Define o espaço **externo** ao redor do elemento, separando-o dos outros elementos da página. A margem não faz parte do tamanho visual do elemento, mas influencia diretamente o layout.

<br />

> [!IMPORTANT]
>
> O tamanho final de um elemento na tela é o resultado da **soma do conteúdo + padding + border**, enquanto a `margin` afeta apenas o espaçamento externo. Entender o Box Model é essencial para criar layouts previsíveis, alinhar elementos corretamente e evitar problemas comuns de espaçamento no CSS.

<br />

<h2>1. Largura e Altura</h2>



As propriedades **width** (largura) e **height** (altura) são responsáveis por definir o tamanho de um elemento no CSS. Elas fazem parte da base do layout e estão diretamente relacionadas ao **box model**, influenciando como os elementos se organizam e ocupam espaço na página.

De forma simples, pense nelas como as medidas de uma caixa:

- **`width`** define o quanto essa caixa ocupa no sentido horizontal;
- **`height`** define o quanto ela ocupa no sentido vertical.

Ao aplicar **width** e **height** a um elemento, você está definindo o tamanho da **área de conteúdo** (content), desconsiderando, por padrão, padding e border.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplos:**

```css
.caixa {
  width: 300px;
  height: 150px;
}
```

Nesse caso, o conteúdo da caixa terá exatamente 300px de largura e 150px de altura.

As propriedades width e height aceitam diferentes tipos de valores, cada um adequado a um contexto específico. Podem usar unidades relativas ao elemento pai ou ao viewport.

- **%:** relativo ao tamanho do elemento pai
- **vw:** relativo à largura da viewport
- **vh:** relativo à altura da viewport

```css
.banner {
  width: 100%;
  height: 50vh;
}
```

Esse exemplo cria um banner que ocupa toda a largura da tela e metade da altura visível do navegador.

O valor padrão é `auto`, o que significa que o navegador calcula o tamanho com base no conteúdo.

```css
.texto {
  width: auto;
  height: auto;
}
```

Esse comportamento é muito comum em textos e elementos inline, que se ajustam naturalmente ao conteúdo.

Diferentemente da largura, a altura costuma depender mais do conteúdo. Em muitos casos, definir apenas a width é suficiente, deixando a height automática.

```css
.card {
  width: 300px;
}
```

Essa prática evita cortes de texto e melhora a adaptação do layout a diferentes tamanhos de conteúdo.

<br />

<h3>1.1. Altura e Largura Máximas e Mínimas</h3>



Definir **larguras** e, principalmente, **alturas fixas** pode gerar comportamentos indesejados, sobretudo em páginas com conteúdo dinâmico ou layouts responsivos. Textos maiores, traduções, dados vindos de APIs ou ajustes de acessibilidade (como aumento de fonte) podem fazer com que o conteúdo ultrapasse os limites do elemento ou seja cortado.

Para evitar esses problemas, o CSS oferece propriedades que permitem **limitar o crescimento ou a redução dos elementos**, sem engessá-los.

As principais propriedades são:

- **`min-height`**: define a altura mínima do elemento
- **`max-height`**: define a altura máxima do elemento
- **`min-width`**: define a largura mínima do elemento
- **`max-width`**: define a largura máxima do elemento

Essas propriedades garantem que o elemento **nunca fique menor ou maior do que o limite definido**, mas ainda permita adaptações conforme o conteúdo ou o tamanho da tela.

```css
.caixa {
  min-height: 150px;
  max-height: 300px;
}
```

No exemplo acima:

- A caixa **nunca terá menos de 150px de altura**, mesmo que o conteúdo seja pequeno.
- A caixa **não ultrapassará 300px de altura**, mesmo que o conteúdo seja maior.

<br />

<h3>1.2. Função de cálculo</h3>



A função de cálculo permite criar valores baseados em **operações matemáticas**, combinando diferentes unidades de medida de forma segura e previsível.

A função `calc()` é usada para realizar cálculos envolvendo unidades distintas, como `%`, `px`, `rem` e `vw`.

```css
.container {
  width: calc(100% - 2rem);
}
```

Nesse exemplo, o elemento ocupa quase toda a largura disponível, mantendo um espaçamento fixo nas laterais. Esse recurso é muito útil para criar layouts fluidos sem perder controle sobre margens e espaçamentos.

<br />

### Boas práticas

- Prefira **larguras relativas** (`%`, `vw`) combinadas com `max-width`.
- Evite **alturas fixas** em elementos com conteúdo variável.
- Utilize `min-height` para preservar o espaçamento visual mínimo.
- Use essas propriedades como **limites**, não como valores fixos principais.

Esse uso torna o layout mais previsível, acessível e adaptável a diferentes cenários, sem comprometer a experiência do usuário.

<br />

<h2>2. Margem e Preenchimento</h2>



As propriedades **`margin`** e **`padding`** são fundamentais para controlar os espaçamentos no CSS. Ambas permitem definir espaço ao redor dos elementos, porém com finalidades diferentes:

- **`margin`** controla o espaço **externo**, separando um elemento dos demais.
- **`padding`** controla o espaço **interno**, entre o conteúdo e a borda do elemento.

<br />

<h3>2.1. Direções disponíveis</h3>



Tanto `margin` quanto `padding` podem ser configuradas individualmente em quatro direções:

- **top**: espaço superior (`margin-top` ou `padding-top`)
- **right**: espaço à direita (`margin-right` ou `padding-right`)
- **bottom**: espaço inferior (`margin-bottom` ou `padding-bottom`)
- **left**: espaço à esquerda (`margin-left` ou `padding-left`)

Essas direções seguem sempre a mesma lógica no CSS.

<br />

<h3>2.2. Forma abreviada (shorthand)</h3>



As propriedades `margin` e `padding` permitem uma escrita abreviada, cujo comportamento varia conforme a quantidade de valores informados:

- **1 valor**
   Aplica o mesmo valor para **todos os lados** (top, right, bottom e left).
- **2 valores**
  - Primeiro valor: **top e bottom** (vertical)
  - Segundo valor: **right e left** (horizontal)
- **3 valores**
  - Primeiro valor: **top**
  - Segundo valor: **right e left**
  - Terceiro valor: **bottom**
- **4 valores**
   Aplicados no sentido **horário**, começando pelo topo:
   **top → right → bottom → left**

Esse padrão é amplamente utilizado no CSS e vale também para outras propriedades, como `border-radius`.

<br />

Já no exemplo a seguir, observamos a aplicação da propriedade **`padding`**, evidenciando o espaçamento interno entre o conteúdo e a borda do elemento:

> Embora `margin` e `padding` tenham sintaxe semelhante, seus efeitos no layout são distintos. Compreender essa diferença é essencial para criar interfaces organizadas, evitar espaçamentos inesperados e manter consistência visual em projetos web.

<h2>3. Borda</h2>



As **bordas** permitem delimitar visualmente os elementos na página, ajudando na organização do layout, na separação de conteúdos e no destaque de componentes importantes.

No CSS, as bordas são compostas por quatro propriedades principais:

- **Estilo** (`border-style`)
- **Espessura** (`border-width`)
- **Cor** (`border-color`)
- **Raio** (`border-radius`)

Cada uma dessas propriedades pode ser configurada **individualmente para cada lado** do elemento (top, right, bottom e left). Além disso, o raio pode ser definido separadamente para **cada canto**, permitindo bordas arredondadas personalizadas.

<br />

<h3>3.1. Estilo</h3>



A propriedade `border-style` define **como a borda será desenhada visualmente**. Sem ela, nenhuma borda será exibida, mesmo que cor e espessura estejam configuradas.

Os valores mais comuns são:

- **solid**: linha contínua ao redor do elemento
- **dashed**: linha formada por traços
- **dotted**: linha formada por pontos
- **double**: duas linhas paralelas
- **hidden**: oculta a borda (comportamento semelhante ao `none`)
- **none**: remove completamente a borda

Outros estilos menos usados, com efeito visual 3D, incluem:

- **groove**: aparência esculpida (oposto do `ridge`)
- **ridge**: aparência em relevo
- **inset**: aparência afundada
- **outset**: aparência elevada

A propriedade `border-width` define a **espessura da borda**, normalmente utilizando unidades como `px`, `rem` ou `em`.

Assim como outras propriedades do Box Model, é possível:

- Definir **um único valor** para todas as bordas

<h2>4. Raio (Arredondamento)</h2>



Todo elemento no CSS possui **quatro extremidades (cantos)**, que podem ser arredondadas individualmente por meio da propriedade `border-radius`:

- **top-left** — canto superior esquerdo
- **top-right** — canto superior direito
- **bottom-right** — canto inferior direito
- **bottom-left** — canto inferior esquerdo

A imagem abaixo ilustra a posição de cada canto:

<div align="center"><img src="https://i.imgur.com/x6ZSllP.png" title="source: imgur.com" /></div>

A propriedade `border-radius` segue a **ordem dos cantos no sentido horário**, iniciando pelo canto superior esquerdo. O comportamento da propriedade varia conforme a quantidade de valores informados:

- **1 valor**
   O mesmo arredondamento é aplicado aos **quatro cantos** do elemento.
- **2 valores**
  - 1º valor → `top-left` (superior esquerdo) e `bottom-right` (inferior direito)
  - 2º valor → `top-right` (superior direito) e `bottom-left` (inferior esquerdo)
- **3 valores**
  - 1º valor → `top-left` (superior esquerdo)
  - 2º valor → `top-right` (superior direito) e `bottom-left` (inferior esquerdo)
  - 3º valor → `bottom-right` (inferior direito)
- **4 valores**
  - 1º valor → `top-left` (superior esquerdo)
  - 2º valor → `top-right` (superior direito)
  - 3º valor → `bottom-right` (inferior direito)
  - 4º valor → `bottom-left` (inferior esquerdo)

Essa flexibilidade permite criar desde cantos levemente arredondados até formatos mais elaborados, como cartões, botões, avatares e componentes visuais modernos.

A imagem abaixo mostra alguns exemplos de uso criativo da propriedade `border-radius`:

<div align="center"><img src="https://i.imgur.com/13hmSP4.png" title="source: imgur.com" /></div>

### Boas práticas de uso

- Utilize valores proporcionais ao tamanho do elemento para manter equilíbrio visual
- Evite exagerar no arredondamento em layouts mais formais
- Combine `border-radius` com `padding` e `background-color` para resultados mais harmônicos

<h2>5. A Propriedade Box Sizing</h2>



Como vimos no **Box Model**, o CSS permite adicionar **preenchimento interno (`padding`)** e **bordas (`border`)** aos elementos. Porém, por padrão, esses valores **são somados à largura e à altura definidas**, o que pode causar problemas de layout.

Esse comportamento padrão ocorre porque o CSS utiliza, por default, o valor:

```css
box-sizing: content-box;
```

Nesse modo, o navegador calcula o tamanho final do elemento da seguinte forma:

- **Altura final** = altura definida + padding (se houver) + borda (se houver)
- **Largura final** = largura definida + padding (se houver) + borda (se houver)

<br />

<h3>5.1. Por que isso é um problema?</h3>



Imagine quatro `<div>` com `width: 25%`. Teoricamente, elas deveriam ocupar exatamente **100% da largura do elemento pai**.
Porém, se qualquer uma dessas caixas receber `padding` ou `border`, seu tamanho final ultrapassará os 25%, fazendo com que as caixas **quebrem de linha ou extrapolem o container**.

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**



Suponha a seguinte configuração:

- Altura definida: **150px**
- Largura definida: **300px**
- Borda: **20px em cada lado**

O cálculo será:

- **Altura final** = 150px + 20px (top) + 20px (bottom) = **190px**
- **Largura final** = 300px + 20px (left) + 20px (right) = **340px**

Ou seja, o elemento cresce **40px no total**, apenas por causa das bordas.

<div align="center"><img src="https://i.imgur.com/oeolfkT.png" title="source: imgur.com" /></div>

Esse aumento inesperado é uma das principais causas de layouts visualmente “quebrados”

<br />

<h3>5.2. Como o `box-sizing` resolve isso?</h3>



A propriedade **`box-sizing`** permite controlar **como o navegador calcula a largura e a altura final de um elemento**.

Ao utilizar:

```css
box-sizing: border-box;
```

o cálculo passa a incluir **padding e borda dentro do tamanho definido**, mantendo a largura e altura finais exatamente como especificadas.

Ou seja:

- A largura definida **já inclui** padding e borda
- A altura definida **já inclui** padding e borda
- O elemento **não cresce além do tamanho definido**

<div align="center"><img src="https://i.imgur.com/R7jXYoP.png" title="source: imgur.com" /></div>

Note que, mesmo com bordas aplicadas, o tamanho total da caixa permanece o mesmo. O espaço das bordas e do padding é “consumido” internamente.

<br />

**Boas práticas**

- Utilize `box-sizing: border-box` como padrão do projeto
- Facilita o cálculo de layouts responsivos
- Evita efeitos colaterais ao adicionar padding e bordas
- Torna o CSS mais previsível e fácil de manter

Uma prática comum é aplicar essa configuração globalmente:

```css
* {
  box-sizing: border-box;
}
```

Assim, todos os elementos da página seguem o mesmo comportamento, reduzindo erros e inconsistências no layout.

<h2>6. Hierarquia de Conteúdo e o Box Model</h2>



Além da hierarquia semântica, um layout bem estruturado depende de um **uso consciente do Box Model**. Espaçamentos previsíveis, larguras controladas e cálculos consistentes evitam que o design “quebre” conforme o conteúdo cresce ou a tela varia de tamanho.

No exemplo abaixo, a hierarquia é reforçada não apenas pelos títulos, mas também por **padding, margin e largura máxima**, garantindo conforto visual e manutenção simples.

<br />

<img src="https://i.imgur.com/O5NjoiA.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<!DOCTYPE html>
<html lang="pt-br">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Página Exemplo</title>
  </head>
  <body>
    <header class="container">
      <h1>Tipografia no Desenvolvimento Web</h1>
      <p>
        Um guia introdutório para entender como organizar textos de forma clara,
        legível e hierárquica em páginas HTML.
      </p>
    </header>

    <main class="container">
      <section class="section">
        <h2>O papel da tipografia</h2>
        <p>
          A tipografia é responsável por tornar o conteúdo compreensível e agradável
          de ler. Uma boa escolha tipográfica ajuda o usuário a entender a mensagem
          sem esforço.
        </p>
        <p>
          Mesmo com um bom conteúdo, uma tipografia mal aplicada pode dificultar a
          leitura e causar abandono da página.
        </p>
      </section>

      <section class="section">
        <h2>Hierarquia visual</h2>
        <p>
          A hierarquia visual orienta o leitor sobre o que é mais importante em uma
          página. Títulos devem chamar mais atenção, enquanto parágrafos devem ser
          confortáveis para leitura contínua.
        </p>
        <p>
          Em HTML, a hierarquia começa com o uso correto das tags de título, como
          <strong>&lt;h1&gt;</strong> e <strong>&lt;h2&gt;</strong>, e é reforçada
          visualmente com CSS.
        </p>
      </section>

      <section class="section">
        <h2>Boas práticas para textos</h2>
        <p>
          Textos longos devem ter espaçamento adequado entre linhas e parágrafos.
          Isso cria pausas naturais e melhora a experiência de leitura.
        </p>
        <p>
          Além disso, limitar a largura do texto evita linhas muito longas, tornando
          a leitura mais fluida e confortável.
        </p>
      </section>
    </main>
  </body>
</html>
```

```css
/* Reset básico para previsibilidade do Box Model */
* {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
}

body {
    font-family: Arial, Helvetica, sans-serif;
    line-height: 1.6;
    background-color: #f9f9f9;
}

/* Container central com largura controlada */
.container {
    max-width: 800px;
    padding: 1.5rem;
    margin: 0 auto;
}

/* Separação lógica entre blocos de conteúdo */
.section {
    margin-top: 2rem;
}

h1 {
    font-size: 2rem;
    margin-bottom: 0.75rem;
}

h2 {
    font-size: 1.4rem;
    margin-bottom: 0.5rem;
}

p {
    margin-bottom: 1rem;
}
```

Observe, no exemplo acima, que a estrutura segue as **boas práticas de semântica, hierarquia, legibilidade e do Box Model**:

- Uso de `box-sizing: border-box` para **evitar cálculos inesperados de largura e altura**.
- Aplicação de um **container com `max-width`**, limitando o comprimento das linhas de texto.
- Uso de `padding` interno no container, separando o conteúdo das bordas da tela.
- Espaçamentos feitos com `margin`, nunca com `<br>` para layout.
- Separação visual entre seções usando `margin-top`, respeitando a hierarquia do conteúdo.
- Layout previsível, legível e adaptável a diferentes tamanhos de tela.
