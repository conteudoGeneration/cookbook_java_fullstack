<h1>Introdução ao CSS</h1>



Quando criamos uma página web, o **HTML** é responsável por **estruturar o conteúdo**, como textos, imagens e links. O **CSS**, por sua vez, é a linguagem utilizada para **definir a apresentação visual** desse conteúdo, controlando cores, fontes, espaçamentos, tamanhos e o posicionamento dos elementos na tela.

O acrônimo **CSS** significa *Cascading Style Sheets* (em português, *Folhas de Estilo em Cascata*). O termo *cascading* (cascata) refere-se à **ordem de prioridade** que o navegador utiliza para aplicar os estilos, levando em conta a **origem das regras**, a **especificidade dos seletores** e a **ordem em que o código é declarado**.

> O termo *Cascading* (cascata) indica que o navegador segue uma ordem de prioridade para aplicar estilos, considerando origem, especificidade e ordem do código.

Criado para separar a **estrutura** da **apresentação**, o CSS tornou o desenvolvimento web mais organizado, reutilizável e fácil de manter. Ele atua exclusivamente sobre a **aparência e o layout** dos elementos, sem alterar o significado ou a estrutura do conteúdo HTML.

> O CSS define exclusivamente a aparência e o layout dos elementos, sem alterar a estrutura ou o significado do conteúdo HTML.

O **CSS** foi **proposto em 1994** por **Håkon Wium Lie**, enquanto trabalhava no **CERN**, o mesmo instituto onde a World Wide Web foi criada. A **primeira especificação oficial**, conhecida como **CSS nível 1 (CSS1)**, foi publicada em **17 de dezembro de 1996** pelo **W3C (World Wide Web Consortium)**, que mantém o CSS até hoje.

Desde então, o CSS evoluiu de forma contínua, passando por especificações como **CSS2** e **CSS3** e, atualmente, adotando um **modelo modular de evolução**. Esse modelo permite a introdução gradual de novos recursos, como **Flexbox**, **Grid Layout**, **animações**, **transições** e **variáveis CSS**, possibilitando a criação de **interfaces modernas, responsivas e acessíveis**, adaptadas a diferentes dispositivos e tamanhos de tela.

Em conjunto com **HTML** e **JavaScript**, o CSS compõe a base do **desenvolvimento Frontend**, sendo indispensável para transformar páginas simples em aplicações web visualmente agradáveis, funcionais e alinhadas aos padrões modernos da web.

<h2>1. Como o CSS funciona no navegador?</h2>



Para entender melhor o papel do CSS, é importante saber **como o navegador interpreta e aplica os estilos** em uma página web. Esse processo acontece em etapas bem definidas, que envolvem a leitura do HTML, do CSS e a renderização visual do conteúdo.

<br />

<h3>1.1. Leitura do HTML e construção do DOM</h3>



Quando uma página é carregada, o navegador começa lendo o arquivo **HTML** e transforma cada tag em uma estrutura de dados chamada **DOM (Document Object Model)**. O DOM representa a página como uma **árvore de elementos**, onde cada nó corresponde a uma tag HTML.

Em termos simples, é como se o navegador criasse um “mapa” da estrutura da página: cabeçalho, seções, parágrafos, imagens, links, entre outros.

<br />

<h3>1.2. Leitura do CSS e construção do CSSOM</h3>



Em paralelo, o navegador lê todos os arquivos **CSS** associados à página (arquivos externos, estilos internos e inline) e constrói outra estrutura chamada **CSSOM (CSS Object Model)**.

O CSSOM contém todas as regras de estilo válidas, já organizadas e prontas para serem aplicadas aos elementos do DOM.

<br />

<h3>1.3. Combinação do DOM com o CSSOM</h3>



Após criar o DOM e o CSSOM, o navegador combina essas duas estruturas para gerar a **árvore de renderização (Render Tree)**.

Nessa etapa:

- Apenas os elementos **visíveis** são considerados;
- Cada elemento recebe os estilos finais que realmente serão aplicados;
- O navegador resolve conflitos de estilo com base na **cascata**, **especificidade** e **ordem das regras**.

<br />

<h3>1.4. A cascata e a prioridade dos estilos</h3>



O CSS segue o princípio da **cascata**, que define como o navegador decidirá qual estilo aplicar quando existem regras conflitantes. A prioridade é determinada principalmente por três fatores:

1. **Origem do estilo**
   - Estilos do navegador (padrão)
   - Estilos do desenvolvedor (CSS externo, interno ou inline)
   - Estilos do usuário (configurações de acessibilidade)
2. **Especificidade do seletor**
   Seletores mais específicos têm prioridade sobre seletores genéricos.
   Exemplo: `#id` tem mais peso que `.classe`, que tem mais peso que `p`.
3. **Ordem do código**
   Quando a especificidade é a mesma, a **última regra declarada** prevalece.

<br />

<h3>1.5. Cálculo de layout (reflow)</h3>



Com a árvore de renderização pronta, o navegador calcula:

- Tamanhos dos elementos;
- Posições na tela;
- Espaçamentos, margens e alinhamentos.

Esse processo é chamado de **layout** ou **reflow**. Ele define exatamente **onde cada elemento será exibido** na página.

<br />

<h3>1.6. Pintura e exibição na tela (paint)</h3>



Após o layout, o navegador realiza a etapa de **pintura (paint)**, desenhando visualmente os elementos na tela:

- Cores;
- Bordas;
- Sombras;
- Textos e imagens.

Por fim, a página é exibida ao usuário.

<br />

<h3>1.7. Atualizações dinâmicas</h3>



Sempre que ocorre uma mudança no HTML ou no CSS (por exemplo, ao passar o mouse em um elemento com `:hover` ou ao alterar classes via JavaScript), o navegador pode:

- Recalcular estilos;
- Reexecutar o layout;
- Repintar partes da tela.

Por isso, escrever CSS bem organizado e eficiente ajuda a melhorar o **desempenho** e a **experiência do usuário**.

<br />

<h2>2. Estrutura básica de uma regra CSS</h2>



Uma **regra CSS** é uma instrução que informa ao navegador **quais elementos HTML devem ser estilizados** e **de que forma** isso será feito.

Toda regra CSS é composta por duas partes principais:

- **Seletor**: define qual ou quais elementos HTML receberão o estilo.
- **Bloco de declarações**: fica entre chaves `{}` e contém uma ou mais declarações de estilo.

Cada **declaração CSS** é formada por:

- uma **propriedade** (o que será alterado);
- um **valor** (como será alterado).

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```css
p {
	font-size: 20px;
}
```

Nesse exemplo, `p` é o **seletor**, que seleciona todas as tags `<p>` da página. A declaração `font-size: 20px;` define que o tamanho da fonte desses parágrafos será de **20 pixels**.

<br />

É possível escrever **várias regras CSS**, cada uma direcionada a diferentes elementos:

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```css
span {
	font-size: 20px;
}

a {
	color: blue;
}
```

<br />

Também é possível **aplicar o mesmo estilo a múltiplos seletores**, separando-os por vírgula:

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```css
p, 
a {
	font-size: 20px;
}
```

Nesse caso, tanto os parágrafos (`<p>`) quanto os links (`<a>`) receberão o mesmo tamanho de fonte.

<br />

<h2>3. Tipos de seletores</h2>



Os **seletores CSS** são utilizados para indicar **quais elementos HTML serão estilizados**. Abaixo estão os principais tipos, usados com mais frequência no desenvolvimento web:

- **Seletores de tag**: aplicam estilos a todas as tags de um mesmo tipo, como `p`, `a`, `h1`.
- **Seletores de classe**: identificam um **grupo de elementos** que compartilham a mesma característica. No HTML são definidos com o atributo `class` e, no CSS, são representados por um ponto (`.`). **Exemplo:** `.minha-classe`.
- **Seletores de ID**: identificam um **elemento único** na página. No HTML utilizam o atributo `id` e, no CSS, são representados por `#`. **Exemplo:** `#meu-id`.

Além dos seletores básicos, o CSS oferece recursos mais avançados para seleções específicas:

- **Pseudoclasses**: permitem aplicar estilos conforme o **estado do elemento**, como `:hover` (quando o mouse passa por cima) ou `:first-child` (primeiro filho).
- **Pseudoelementos**: criam partes “virtuais” do elemento, como `::before` e `::after`, muito usados para efeitos visuais ou inserção de conteúdo sem alterar o HTML.

Esses seletores ampliam o controle visual da página e permitem criar interações e efeitos apenas com CSS.

Mais adiante, teremos um capítulo inteiro dedicado aos seletores CSS.

<br />

<h2>4. Regras Obrigatórias</h2>



Para garantir o funcionamento correto do CSS, algumas regras básicas devem ser seguidas:

- Cada declaração CSS deve terminar com **ponto e vírgula (`;`)**.
- Mesmo na **última declaração** do bloco, o ponto e vírgula deve ser mantido, pois isso evita erros e facilita a manutenção e a leitura do código.

<br />

<h2>5. Boas práticas de formatação</h2>



Embora o CSS funcione mesmo sem uma formatação organizada, adotar boas práticas torna o código mais legível, profissional e fácil de manter.

Algumas recomendações importantes:

- Mantenha o seletor alinhado à esquerda.
- Abra as chaves `{}` na mesma linha do seletor.
- Indente as declarações internas usando espaços ou tabulação.
- Utilize sempre o mesmo padrão de indentação ao longo do projeto.

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```css
p {
  font-size: 20px;
  color: #333;
}

a:hover {
  text-decoration: underline;
}
```

Uma boa formatação facilita a leitura do código, reduz erros e torna o trabalho em equipe mais eficiente.

<br />

<h2>6. Adicionando CSS em uma Página HTML</h2>



Existem **três formas principais** de adicionar CSS a uma página web. Cada uma delas possui finalidades específicas e deve ser utilizada de acordo com o contexto do projeto.

Compreender essas abordagens é essencial para organizar corretamente o código e facilitar sua manutenção.

<br />

<h3>6.1.  Arquivo externo com a tag link </h3>



Essa é a **forma mais recomendada** de utilizar CSS, especialmente em projetos com várias páginas. 

Nesse método, as regras de estilo ficam em um arquivo separado com a extensão `.css`, que é vinculado ao HTML por meio da tag `<link>`, localizada dentro da tag `<head>`.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="icon" href="./assets/img/favicon.svg" />

    <!-- Links para as Folhas de Estilo da Página - CSS -->
    <link rel="stylesheet" href="./assets/css/style.css">

    <title>Minha Página</title>
</head>
```

**Vantagens desse método:**

- Mantém o HTML mais limpo e organizado.
- Facilita a manutenção do código.
- Permite reutilizar o mesmo arquivo CSS em várias páginas.
- Segue as boas práticas do desenvolvimento web.

Por esses motivos, essa abordagem é a mais utilizada em projetos profissionais.

<br />

<h3>6.2. Estilos internos com a tag style</h3>



Outra opção é escrever as regras CSS diretamente no arquivo HTML, utilizando a tag `<style>`, também dentro do `<head>` da página.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<style>
	p {
		font-size: 20px;
    }

    a {
		color: blue;
    }
</style>
```

Esse método pode ser útil em situações específicas, como:

- Testes rápidos de estilo.
- Páginas simples ou isoladas.
- Exemplos didáticos.

No entanto, **não é recomendado para projetos maiores**, pois mistura a estrutura HTML com a estilização, dificultando a manutenção e a reutilização do código.

<br />

<h3>6.3. Estilos embutidos (inline)</h3>



A terceira forma de aplicar CSS é por meio de **estilos embutidos**, utilizando o atributo `style` diretamente no elemento HTML.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<div style="background-color: yellow">
    Conteúdo da Div
</div>
```

Nesse caso, o estilo afeta apenas aquele elemento específico.

Apesar de funcionar, esse método deve ser usado **com muita moderação**, pois:

- Dificulta a reutilização dos estilos.
- Torna o código mais difícil de manter.
- Vai contra o princípio de separação entre estrutura e apresentação.

Por isso, o uso de estilos inline é geralmente restrito a casos muito pontuais.

<br />

<h2>7. Comentários</h2>



Os **comentários em CSS** são trechos de texto inseridos no código que **não afetam a estilização da página** e **não são interpretados pelo navegador**. Eles são utilizados para:

- Explicar regras ou grupos de estilos;
- Organizar o código por seções (cores, layout, tipografia, etc.);
- Deixar anotações para outros desenvolvedores (ou para você mesmo no futuro);
- Desativar temporariamente regras durante testes e ajustes visuais.

O uso de comentários é uma **boa prática**, pois melhora a **legibilidade**, a **organização** e a **manutenção** das folhas de estilo, especialmente em projetos maiores.

**Sintaxe:**

```css
/* Este é um comentário */
```

**Exemplo:**

```css
/* Estilos para parágrafos */
p {
  font-size: 16px;
  color: #333;
}
```

Diferente do HTML, o CSS utiliza **apenas um tipo de comentário**, válido tanto para **uma única linha** quanto para **múltiplas linhas**, o que facilita a padronização e a organização do código.
