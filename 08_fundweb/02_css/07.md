<h1>Estilização - Background (Plano de Fundo)</h1>



No CSS, é possível **alterar o plano de fundo de um elemento** de diferentes maneiras, utilizando **cores sólidas**, **gradientes** ou **imagens**. Esses recursos permitem criar desde layouts simples até interfaces visuais mais elaboradas e modernas.

A seguir, serão apresentadas as **principais formas de estilização de fundo**, com **exemplos práticos**, facilitando a compreensão e a aplicação desses conceitos em projetos reais.

Para demonstrar as estilizações, vamos utilizar o código HTML abaixo:

<img src="https://i.imgur.com/O5NjoiA.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<!DOCTYPE html>
<html lang="pt-br">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Página Exemplo</title>
  </head>
  <body>
    <div>
      <h1>Título do Texto</h1>
      <h2>Subtítulo do texto</h2>
      <p>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. Pellentesque

      </p>
    </div>
  </body>
</html>

```

<h2>1. Cor Sólida</h2>



A propriedade básica para definir a cor de fundo de um elemento HTML é `background-color`. Ela pode aceitar diferentes formatos de cores, como:

- Nomes de cores ( `red`, `blue`)
- Códigos hexadecimais ( `#ff0000`)
- Formatos RGB e RGBA ( `rgb(255, 0, 0)`, `rgba(255, 0, 0, 0.5)`)
- entre outros

<br />

**Exemplo:**

```css
div {
  background-color: #cccccc; /* Cor Cinza hexadecimal */
}
```

**Resultado:**

![](https://i.imgur.com/AxzOuHk.png)

No exemplo acima estamos aplicando a cor cinza no plano de fundo de todas as div’s do site.

<h2>2. Gradiente</h2>



O **Gradiente (gradient)** em CSS é uma forma de criar **transições suaves entre duas ou mais cores**, sem a necessidade de usar imagens externas. Em vez de uma cor sólida, o navegador renderiza uma mistura gradual de cores diretamente pelo código CSS.

Na prática, o gradiente é tratado como um **tipo especial de imagem**, gerada dinamicamente pelo navegador. Por isso, ele é usado principalmente em propriedades como `background` ou `background-image`.

Um exemplo simples do dia a dia: imagine um céu que começa azul escuro no topo e vai clareando até o horizonte. Esse efeito visual é exatamente o que um degradê representa.

O uso de gradientes em CSS possui algumas características importantes:

- **Não é uma cor**, e sim uma imagem gerada por código
- **Dispensa arquivos externos**, como PNG ou JPG
- **É responsivo**, adaptando-se automaticamente ao tamanho do elemento
- **Pode ter duas ou várias cores**
- **Permite controle de direção, ângulo e pontos de parada das cores**

Essas características tornam o degradê muito útil para fundos de páginas, botões, banners, cards e destaques visuais.

<br />

<h3>2.1. Tipos de gradientes em CSS</h3>



O CSS oferece dois tipos principais de gradientes:

<br />

### a) **Gradiente Linear**



O gradiente linear muda de uma cor para outra ao longo de uma linha, que pode ser definida em diferentes direções (vertical, horizontal, diagonal, entre outras).

**Exemplo mental:** uma folha pintada de cima para baixo, começando azul e terminando roxa.

<br />

**Sintaxe básica:**

```css
div {
  background: linear-gradient(to right, #c6edff, #a3c6ff);
}
```

**Resultado:**

![](https://i.imgur.com/8FfBxK4.png)

No exemplo acima estamos aplicando um gradiente na cor de fundo, do tipo linear, que inicia a partir do lado direito (`to right`) com um tom de azul mais claro até chegar em um tom de azul um pouco mais escuro.

Na tabela abaixo, temos uma lista com as **direções possíveis de um gradiente linear em CSS**, considerando tanto as **direções semânticas** quanto os **ângulos em graus**, que são equivalentes entre si.

### Direções de Gradiente Linear (`linear-gradient`)

| Direção (palavra-chave) | Ângulo equivalente | Descrição do sentido do gradiente                  |
| ----------------------- | ------------------ | -------------------------------------------------- |
| `to top`                | `0deg`             | De baixo para cima                                 |
| `to top right`          | `45deg`            | Do canto inferior esquerdo para o superior direito |
| `to right`              | `90deg`            | Da esquerda para a direita                         |
| `to bottom right`       | `135deg`           | Do canto superior esquerdo para o inferior direito |
| `to bottom`             | `180deg`           | De cima para baixo                                 |
| `to bottom left`        | `225deg`           | Do canto superior direito para o inferior esquerdo |
| `to left`               | `270deg`           | Da direita para a esquerda                         |
| `to top left`           | `315deg`           | Do canto inferior direito para o superior esquerdo |

### Observações importantes

- O **valor padrão** do `linear-gradient` é `to bottom` (`180deg`), caso nenhuma direção seja informada.
- Ângulos em CSS começam em **0deg apontando para cima** e aumentam no **sentido horário**.
- Usar palavras-chave (`to right`, `to bottom`, entre outras.) melhora a **legibilidade** do código, enquanto ângulos oferecem **maior precisão** em layouts mais complexos.

<br />

### b) **Gradiente Radial**



O gradiente radial espalha-se de um ponto central, criando uma transição em forma de círculo ou elipse.

**Exemplo mental:** uma lanterna acesa no escuro, com a luz mais forte no centro e mais fraca nas bordas.

<br />

**Sintaxe básica:**

```css
div {
  background: radial-gradient(circle, #c6edff, #a3c6ff)
}
```

**Resultado:**

![](https://i.imgur.com/qrh5M10.png)

No exemplo acima estamos aplicando um gradiente na cor de fundo, do tipo radial circular, que inicia a partir do centro com uma cor azul no tom claro até chegar na cor azul no tom mais escuro.

Na tabela abaixo, temos uma lista com os **principais tipos e variações do gradiente radial no CSS**, organizados de forma clara para facilitar o entendimento e o uso no dia a dia.

### Tipos de Gradiente Radial (`radial-gradient`)

| Tipo / Parâmetro          | Valor CSS         | Descrição                                                    |
| ------------------------- | ----------------- | ------------------------------------------------------------ |
| **Forma padrão**          | `ellipse`         | Forma elíptica (oval). É o valor padrão do gradiente radial. |
| **Forma circular**        | `circle`          | Forma perfeitamente circular.                                |
| **Tamanho padrão**        | `farthest-corner` | O gradiente se estende até o canto mais distante do elemento. |
| **Mais próximo**          | `closest-side`    | O gradiente vai até o lado mais próximo do elemento.         |
| **Lado mais distante**    | `farthest-side`   | O gradiente vai até o lado mais distante do elemento.        |
| **Canto mais próximo**    | `closest-corner`  | O gradiente vai até o canto mais próximo do elemento.        |
| **Canto mais distante**   | `farthest-corner` | O gradiente vai até o canto mais distante do elemento (valor padrão). |
| **Posição central**       | `at center`       | Centraliza o gradiente no elemento.                          |
| **Posição personalizada** | `at top left`     | Define o início do gradiente em uma posição específica.      |
| **Posição com valores**   | `at 30% 40%`      | Define a posição inicial usando porcentagens ou medidas.     |

<br />

### Boas práticas

- Use `circle` quando quiser **simetria visual**, como botões ou ícones.
- Prefira `ellipse` para **fundos amplos**, pois se adapta melhor a diferentes proporções de tela.
- Ajustar o tamanho (`closest-*` ou `farthest-*`) ajuda a criar **efeitos de foco e profundidade**.
- Combine gradientes radiais com **cores suaves** para evitar poluição visual.

Essas variações tornam o `radial-gradient` extremamente versátil para criar efeitos modernos sem o uso de imagens.

<h3>2.2. Boas práticas</h3>



- Use **poucas cores** no início (duas já são suficientes)
- Prefira degradês **sutis**, especialmente em fundos
- Teste o contraste para garantir boa legibilidade do texto
- Evite exagerar: o degradê deve complementar o layout, não competir com o conteúdo

## 3. **Imagens de Fundo**



Por meio da propriedade `background-image`, é possível definir uma imagem como plano de fundo de um elemento. Além disso, o CSS oferece outras propriedades que permitem **controlar repetição, tamanho, posição e comportamento da imagem**.

Nos exemplos a seguir, utilizaremos duas imagens e aplicaremos o plano de fundo diretamente no elemento `body`.

| <img src="https://imgur.com/Fq6kGS8.jpg"> | <img src="https://i.imgur.com/2sNhtnn.jpg" height="50%"> |
| ----------------------------------------- | -------------------------------------------------------- |

<br />

### a) **Adicionar uma Imagem de Fundo**

Para definir uma imagem de fundo, basta informar a URL da imagem:

```css
body {
  background-image: url('https://i.imgur.com/Fq6kGS8.jpg');
}
```

**Resultado:**

![](https://i.imgur.com/1yOjIcF.png)

A imagem será exibida, porém apenas uma parte dela aparecerá na tela. Isso ocorre porque a primeira imagem é maior do que a área visível do `body`.

Ao utilizar a segunda imagem, que é menor do que a área visível do `body`:

```css
body{
   background-image: url('https://i.imgur.com/F199BA7.jpg');
}
```

**Resultado:**

![](https://i.imgur.com/i3XNKCG.png)

A imagem se repete automaticamente para preencher o espaço disponível. Esse comportamento é o padrão do CSS.

<br />

### b) **Controlar Repetição de Imagem**

A propriedade `background-repeat` define se a imagem será repetida ou não.

Valores mais comuns:

- `repeat` (padrão)
- `no-repeat`
- `repeat-x`
- `repeat-y`

```css
body{
   background-image: url('https://i.imgur.com/F199BA7.jpg');
   background-repeat: no-repeat;
}
```

**Resultado:**

![](https://i.imgur.com/1HWKa1A.png)

Utilizando a propriedade `backgound-repeat: no-reapeat` na segunda imagem, ela será exibida apenas uma vez. Note que, nesse caso, ela não ocupa toda a área da página.

<br />

### c) **Ajustar Tamanho da Imagem**

A propriedade `background-size` controla como a imagem será dimensionada.

**Valores comuns:**

- `auto`:  valor padrão. A imagem mantém seu **tamanho original**, respeitando suas dimensões naturais
- `cover`: preenche todo o elemento, podendo cortar partes da imagem
- `contain`: ajusta a imagem ao espaço disponível, sem cortes

No caso da primeira imagem, podemos utilizar a opção cover para ajustar a imagem ao body, porque como a imagem é grande e possui uma boa resolução, ela se ajustará sem maiores perdas:

```css
body {
  background-image: url('https://i.imgur.com/Fq6kGS8.jpg');
  background-repeat: no-repeat;
  background-size: cover;
}
```

**Resultado:**

![](https://i.imgur.com/8FgRv7Y.png)

Observe que a imagem ocupa toda a tela com boa qualidade visual.

No caso da segunda imagem até poderíamos usar a opção cover, entretanto, como a imagem é menor, é quase certo que acontecerão perdas e distorções na qualidade da imagem:

```css
body{
   background-image: url('https://i.imgur.com/F199BA7.jpg');
   background-repeat: no-repeat;
   background-size: cover;
}
```

**Resultado:**

![](https://i.imgur.com/irozank.png)

A imagem sofre perda de qualidade e parece estar esticada.

<br />

### d) **Posicionar Imagem**

Com `background-position`, é possível definir onde a imagem será posicionada dentro do elemento.

**Valores comuns:**

- `center`
- `top`, `bottom`
- `left`, `right`
- Combinações como `top right`
- Valores personalizados (`50% 30%`, por exemplo)

```css
body{
   background-image: url('https://i.imgur.com/F199BA7.jpg');
   background-repeat: no-repeat;
   background-size: contain;
   background-position: center;
}
```

**Resultado:**

![](https://i.imgur.com/7v8TlG3.png)

Aplicando a propriedade `background-position: center` na segunda imagem, ela ficará centralizada, ajustada ao espaço disponível e sem distorções.

<br />

### e) **Imagem de Fundo Fixa**

A propriedade `background-attachment` define se a imagem acompanha ou não a rolagem da página.

**Valores comuns:**

- `scroll`: A imagem de fundo rola junto com o conteúdo (comportamento padrão)
- `fixed`: A imagem fica fixa em relação à viewport, criando efeito “parallax simples”
- `local`: A imagem rola junto com o conteúdo **do próprio elemento**

```css
body {
  background-image: url('https://i.imgur.com/Fq6kGS8.jpg');
  background-repeat: no-repeat;
  background-size: cover;
  background-attachment: fixed;
}
```

**Resultado:**

![](https://i.imgur.com/TdxjnGT.png)

Aplicando propriedade `background-attachment` na primeira imagem, ela permanece fixa enquanto o conteúdo da página rola sobre ela, criando um efeito de profundidade.

<br />

<h3>3.1. Dicas Importantes para trabalhar com imagens no Plano de fundo</h3>



1. **Evite utilizar imagens no plano de fundo sempre que possível.** Imagens aplicadas via `background-image` dificultam o controle de responsividade, principalmente em dispositivos móveis, além de impactarem negativamente o desempenho da página.
2. **Utilize imagens de fundo exclusivamente com finalidade decorativa.** Nunca use `background-image` para transmitir informação ou conteúdo relevante, pois essas imagens não possuem valor semântico para mecanismos de busca e tecnologias assistivas.
3. **Prefira imagens otimizadas e formatos modernos.** Utilize compressão adequada e dê preferência a formatos como **WebP** ou **AVIF**, que oferecem melhor qualidade visual com menor tamanho de arquivo.
4. **Evite imagens grandes em páginas iniciais ou acima da dobra.** Imagens pesadas aumentam o tempo de carregamento inicial e prejudicam métricas de performance importantes para SEO, como o **Largest Contentful Paint (LCP)**, ou seja, o tempo que demora para carregar o maior conteúdo visível da página.
5. **Sempre que a imagem fizer parte do conteúdo, utilize a tag `<img>` com atributo `alt`.** Isso garante melhor indexação pelos mecanismos de busca, acessibilidade para leitores de tela e conformidade com boas práticas de HTML semântico.
6. **Não abuse de transparências, filtros ou marcas d’água.** Esses recursos podem comprometer a legibilidade do conteúdo, causar poluição visual e aumentar o peso da imagem desnecessariamente.
7. **Escolha imagens com boa resolução, mas equilibradas.** A imagem deve ter qualidade suficiente para diferentes tamanhos de tela, sem exagerar na resolução a ponto de comprometer a performance.
8. **Evite imagens pequenas ou no formato retrato como plano de fundo.** Dê preferência a imagens grandes, no formato paisagem, que se adaptam melhor a telas horizontais e reduzem cortes e distorções ao usar propriedades como `cover`.

Essas práticas ajudam a manter um layout visualmente agradável, acessível e alinhado aos critérios modernos de desempenho e SEO.
