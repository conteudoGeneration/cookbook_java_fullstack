<h1>Estilização - Texto</h1>



Vimos anteriormente que a **tipografia** refere-se à forma como os textos são **apresentados visualmente** em uma interface: tipo de fonte, tamanho, espaçamento, peso, alinhamento e legibilidade.

No CSS, a tipografia é responsável por transformar o texto “cru” do HTML em algo **agradável, legível e coerente** com o objetivo da página. Um mesmo conteúdo pode transmitir sensações completamente diferentes apenas mudando a tipografia.

No dia a dia, pense na diferença entre um texto de livro, uma placa de trânsito e um site infantil. O conteúdo pode ser semelhante, mas a tipografia muda totalmente a percepção.

Na prática, o **HTML** define o conteúdo textual e sua semântica (`<h1>`, `<p>`, `<strong>`, entre outras), enquanto o **CSS** define **como esse texto será exibido**.

Essa separação é fundamental para manter código organizado e acessível.

Na sequência, veremos como podemos formatar o texto visível em nossas páginas HTML.

<br />

<h2>1. Propriedades de Estilização de Texto</h2>

<br />

<h3>1.1. Famílias de fontes: `font-family`</h3>



Como vimos anteriormente, a propriedade `font-family` define **qual fonte será usada** no texto.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
body {
  font-family: Arial, Helvetica, sans-serif;
}
```

Aqui, o navegador tenta usar:

1. Arial
2. Se não existir, Helvetica
3. Caso nenhuma esteja disponível, uma fonte genérica sem serifa

Isso é chamado de **pilha de fontes** (*font stack*), uma prática essencial para garantir compatibilidade entre dispositivos.

<br />

<h3>1.2. Cor da fonte: `color`</h3>



Da mesma forma que é possível definir **cores de fundo** para os elementos da página por meio das propriedades de *background*, o CSS também permite controlar a **cor do texto** exibido na interface. Para isso, utilizamos a propriedade **`color`**, responsável por definir a cor das fontes de um elemento.

Essa propriedade pode ser aplicada a qualquer elemento que contenha texto, como `body`, `p`, `h1`, `span`, `a`, entre outros. Quando definida em um elemento pai, a cor do texto é **herdada** pelos elementos filhos, a menos que outra regra sobrescreva esse valor.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
html {
	color: #ffffff; /* white */
	background-color: #000000; /* black */
}
```

No exemplo acima, estamos definindo a cor de fundo da página como preto e a cor da letra padrão da página (fonte) como branco:

<div align="center"><img src="https://i.imgur.com/cDXmlbf.png" title="source: imgur.com" /></div>

<br />

### Boas práticas

- Utilize cores com **bom contraste** em relação ao fundo, melhorando a legibilidade
- Prefira definir a cor do texto em um nível mais alto da hierarquia (como no `body`)
- Evite cores muito claras em fundos claros ou muito escuras em fundos escuros
- Considere acessibilidade e leitura em diferentes dispositivos

A propriedade `color` é um dos pilares da estilização tipográfica e deve ser usada em conjunto com uma boa escolha de fontes e espaçamentos para garantir uma experiência de leitura agradável.

<br />

<h3>1.3. Tamanho do texto: font-size</h3>



A propriedade <code>font-size</code> define o **tamanho do texto** exibido na página. Ela influencia diretamente a **legibilidade**, a **hierarquia visual** e a **acessibilidade** do conteúdo.

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
p {
  font-size: 16px;
}
```

No exemplo acima, o texto do parágrafo terá tamanho fixo de **16 pixels**, que corresponde ao tamanho padrão de fonte utilizado pela maioria dos navegadores.

O CSS permite definir o tamanho do texto usando **unidades absolutas** e **unidades relativas**, como vemos na tabela abaixo:

| Unidade | Tipo     | Base de cálculo                | Vantagens                               | Desvantagens                                      | Quando usar                        |
| ------- | -------- | ------------------------------ | --------------------------------------- | ------------------------------------------------- | ---------------------------------- |
| `px`    | Absoluta | Tamanho fixo em pixels         | Simples de entender, previsível         | Pouco flexível, não escala bem com acessibilidade | Projetos simples, ajustes pontuais |
| `em`    | Relativa | `font-size` do elemento pai    | Flexível, permite escalas locais        | Pode causar efeito cascata difícil de controlar   | Componentes isolados               |
| `rem`   | Relativa | `font-size` do elemento `html` | Previsível, fácil manutenção, acessível | Depende da configuração do `html`                 | Layouts modernos e responsivos     |

<br />

### Boas práticas

- Prefira **`rem`** para tamanhos de texto, pois facilita a manutenção e melhora a acessibilidade.
- Utilize **`px`** apenas em casos pontuais ou projetos simples.
- Defina um **tamanho base no `html` ou no `body`** e construa a hierarquia tipográfica a partir dele.
- Evite misturar muitas unidades diferentes sem necessidade.

Uma estratégia comum é definir o tamanho base no `body` e criar variações proporcionais nos títulos e parágrafos, garantindo **consistência visual**, **responsividade** e **facilidade de ajustes futuros**.

<br />

<h3>1.4. Peso da fonte: font-weight</h3>



O `font-weight` controla o **nível de espessura** do texto.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
h1 {
  font-weight: 700;
}
```

**Valores comuns:**

| Peso Numérico | Nome do Peso     | Uso Comum / Indicação                                 |
| ------------- | ---------------- | ----------------------------------------------------- |
| 100           | Thin             | Textos muito leves, uso decorativo ou títulos grandes |
| 200           | Extra Light      | Interfaces minimalistas, textos secundários           |
| 300           | Light            | Textos longos com visual mais suave                   |
| 400           | Regular / Normal | **Peso padrão para textos corridos**                  |
| 500           | Medium           | Destaques sutis, menus ou subtítulos                  |
| 600           | Semi Bold        | Ênfase moderada, botões ou títulos intermediários     |
| 700           | Bold             | **Destaque forte**, títulos, chamadas de ação         |
| 800           | Extra Bold       | Títulos de grande impacto                             |
| 900           | Black / Heavy    | Uso pontual, títulos muito destacados ou branding     |

### Observações importantes

- Nem todas as fontes possuem **todos os pesos disponíveis**
- Se um peso não existir, o navegador utilizará o peso mais próximo
- Sempre verifique no **Google Fonts** ou na documentação da fonte quais pesos estão disponíveis
- Para textos longos, os pesos **400 (regular)** e **500 (medium)** oferecem melhor legibilidade
- Pesos muito altos devem ser usados com moderação para evitar poluição visual

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
h1 {
  font-weight: 700;
}

p {
  font-weight: 400;
}
```

Essa abordagem garante consistência visual, melhora a leitura e facilita a manutenção do projeto.

<br />

<h3>1.5. Estilo da fonte: font-style</h3>



A propriedade `font-style` define variações como o **itálico**.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
p {
  font-style: italic;
}
```

Esta propriedade possui 3 valores:

- `italic`
- `oblique` 
- `normal`

<br />

> [!IMPORTANT]
>
> O valor `italic` utiliza um desenho específico da fonte, enquanto `oblique` apenas inclina artificialmente a fonte original. Em muitas fontes modernas, o efeito visual é semelhante.

<br />

<h3>1.6. Alinhamento do texto: `text-align`</h3>



A propriedade `text-align` define como o texto será alinhado dentro do elemento.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
p {
  text-align: left;
}
```

**Valores comuns:**

- `left` → padrão para leitura
- `center` → títulos e destaques
- `right` → usos pontuais
- `justify` → usado em artigos e textos longos

<br />

<h3>1.7. Transformação do texto: `text-transform`</h3>



A propriedade `text-transform` altera visualmente o texto, sem mudar o conteúdo original.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
button {
  text-transform: uppercase;
}
```

**Valores comuns:**

- `none` → mantém o texto original (valor padrão)
- `uppercase` → tudo em maiúsculas
- `lowercase` → tudo em minúsculas
- `capitalize` → primeira letra de cada palavra

É muito usada em botões e rótulos.

<br />

<h3>1.8. Decoração do texto: `text-decoration`</h3>



A propriedade `text-decoration` é utilizada para **adicionar ou remover decorações visuais** no texto, como sublinhados, linhas sobre o texto ou linhas atravessadas. Ela é muito comum em **links**, **textos destacados** e **estados visuais** de interação.

Por padrão, navegadores aplicam sublinhado em links (<a>). Com CSS, esse comportamento pode ser **personalizado ou removido**, de acordo com o design da interface.

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
a {
  text-decoration: none;
}
```

No exemplo acima, o sublinhado padrão dos links é removido, prática comum em menus e botões estilizados.

<br />

### Valores comuns

| Valor          | Descrição                                        | Uso comum                         |
| -------------- | ------------------------------------------------ | --------------------------------- |
| `none`         | Remove qualquer decoração do texto               | Links, botões                     |
| `underline`    | Adiciona uma linha abaixo do texto               | Links, destaques                  |
| `overline`     | Adiciona uma linha acima do texto                | Uso decorativo                    |
| `line-through` | Adiciona uma linha atravessando o texto (taxado) | Preços antigos, itens desativados |

<br />

### Boas práticas

- Evite remover o sublinhado de links sem fornecer **outro indicativo visual** (cor, ícone ou hover)
- Prefira `text-decoration: none` apenas em links que se comportam como botões
- Utilize `line-through` para representar valores inválidos, antigos ou descontinuados
- Use `overline` com moderação, pois é pouco comum em interfaces

<br />

<h3>1.9. Altura da linha: `line-height`</h3>



A propriedade `line-height` controla o **espaçamento vertical entre linhas de texto**, sendo um dos fatores mais importantes para a legibilidade.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
.lh1{
  line-height: 1;
}

.lh2{
  line-height: 1.6;
}

.lh3{
  line-height: 2;
}
```

<img src="https://i.imgur.com/3HPZ0By.png" title="source: imgur.com" width="3%"/>**Resultado:**

<div align="center"><img src="https://imgur.com/y6WcnzI.png" title="source: imgur.com"/></div>

Observe que, no primeiro parágrafo, as linhas estão muito próximas, o que torna a leitura do texto cansativa. Já no segundo e no terceiro parágrafos, há um maior “respiro” entre as linhas, proporcionando uma leitura mais confortável.

Cada parágrafo recebeu uma classe diferente (`lh1, lh2, lh3`), com um valor distinto na propriedade **line-height**. Dessa forma, o espaçamento entre as linhas do parágrafo varia, pois cada linha passa a ter uma altura diferente. 

O valor informado na propriedade **line-height** funciona como um **multiplicador do valor definido em font-size**.

O tamanho padrão da fonte dos navegadores, por exemplo, é **16px**. Ao utilizar o valor **1.6** na propriedade`line-height`, o cálculo será:

- `16 × 1.6 = 25.6px`  
- `(25.6 − 16) / 2 = 4.8px`  

Isso significa que será adicionado um espaço de **4.8px acima e 4.8px abaixo do texto**, mantendo o conteúdo centralizado verticalmente dentro da linha, garantido a legibilidade do texto.

<div align="center"><img src="https://i.imgur.com/MOhJSrz.png" title="source: imgur.com" /></div>

<br />

**Vantagens:**

- Escala automaticamente se o tamanho da fonte mudar
- Melhor para responsividade
- Mais fácil de manter

<br />

<h3>1.10. Espaçamento entre letras: `letter-spacing`</h3>



A propriedade `letter-spacing` ajusta o **espaço entre os caracteres**.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
h1{
  letter-spacing: 2px;
}

h2{
  letter-spacing: 1px;
}
```

<img src="https://i.imgur.com/3HPZ0By.png" title="source: imgur.com" width="3%"/>**Resultado:**

<div align="center"><img src="https://i.imgur.com/4O5q800.png" title="source: imgur.com"/></div>

Observe que, no Título 01, o espaçamento entre as letras é maior. Já no Título 02, esse espaçamento é um pouco menor, enquanto, no parágrafo, o espaçamento segue o padrão da fonte.

<br />

<h3>1.11. Espaço e quebra de texto</h3>



Além do tamanho, peso e espaçamento entre linhas, o CSS oferece propriedades específicas para **controlar como o texto se comporta dentro de um elemento**, especialmente quando há pouco espaço disponível ou palavras muito longas. As principais são:
`white-space`, `word-break` e `overflow-wrap`.

Essas propriedades são essenciais para garantir **boa legibilidade**, **layout consistente** e evitar que o texto “quebre” o design da página.

<br />

<h4>1.11.1. Controle de espaços e quebras de linha: <code>white-space</code></h4>



A propriedade `white-space` define **como espaços em branco, quebras de linha e quebras automáticas de texto** devem ser tratadas pelo navegador.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
p {
  white-space: normal;
}
```

### Valores mais comuns

| Valor      | Comportamento                                             |
| ---------- | --------------------------------------------------------- |
| `normal`   | Comprime espaços e quebra linhas automaticamente (padrão) |
| `nowrap`   | Não permite quebra de linha                               |
| `pre`      | Mantém espaços e quebras exatamente como no HTML          |
| `pre-wrap` | Mantém espaços, mas permite quebra de linha               |
| `pre-line` | Comprime espaços, mas respeita quebras de linha           |

### Uso prático

- `normal`: textos comuns
- `nowrap`: menus, botões, rótulos curtos
- `pre` / `pre-wrap`: códigos, poemas ou textos formatados

<br />

<h4>1.11.2. Quebra de palavras: <code>word-break</code></h4>



A propriedade `word-break` controla **como as palavras são quebradas quando não cabem no espaço disponível**.

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
p {
  word-break: break-word;
}
```

### Valores principais

| Valor       | Descrição                                   |
| ----------- | ------------------------------------------- |
| `normal`    | Comportamento padrão                        |
| `break-all` | Quebra a palavra em qualquer ponto          |
| `keep-all`  | Evita quebras (útil para idiomas asiáticos) |

<br />

> [!WARNING]
>
> O valor `break-all` deve ser usado com cuidado, pois pode prejudicar a leitura ao quebrar palavras de forma abrupta.

<br />

<h4>1.11.3. Quebra segura de palavras longas: <code>overflow-wrap</code></h4>



A propriedade `overflow-wrap` (antigamente conhecida como word-wrap) define se palavras muito longas podem ser quebradas para evitar **estouro do layout**.

<br />

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
p {
  overflow-wrap: break-word;
}
```

### Valores mais utilizados

| Valor        | Descrição                            |
| ------------ | ------------------------------------ |
| `normal`     | Não força a quebra                   |
| `break-word` | Quebra palavras longas se necessário |

Essa é a **abordagem mais recomendada** para lidar com URL's ou textos sem espaços.

<br />

### Boas práticas

- Use `white-space` para controlar **espaços e quebras de linha**
- Prefira `overflow-wrap: break-word` para evitar problemas de layout
- Evite `word-break: break-all` em textos longos

Essas propriedades garantem que o texto se adapte corretamente ao espaço disponível, mantendo o layout organizado e a leitura confortável, mesmo em telas menores ou conteúdos imprevisíveis.

<h2>2. Hierarquia de Conteúdo</h2>



A **hierarquia de conteúdo** é um dos pilares de uma boa estrutura HTML e de um design legível. Ela define a ordem de importância das informações, orientando tanto o usuário quanto os mecanismos de busca sobre o que é mais relevante na página. Uma hierarquia bem construída melhora a **experiência de leitura**, a **acessibilidade** e o **SEO**, além de facilitar a manutenção do código.

No exemplo abaixo, é possível observar como a hierarquia é aplicada corretamente por meio do uso adequado dos títulos, parágrafos e espaçamentos, criando uma estrutura clara, semântica e fácil de compreender, mesmo antes da aplicação dos estilos em CSS.

<br />

<img src="https://i.imgur.com/O5NjoiA.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<!DOCTYPE html>
<html lang="pt-br">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Página Exemplo</title>
  </head>
  <body>
	<header>
        <h1>Tipografia no Desenvolvimento Web</h1>
        <p>
          Um guia introdutório para entender como organizar textos de forma clara,
          legível e hierárquica em páginas HTML.
        </p>
      </header>

      <main>
        <section>
          <h2>O papel da tipografia</h2>
          <p>
            A tipografia é responsável por tornar o conteúdo compreensível e agradável
            de ler. Uma boa escolha tipográfica ajuda o usuário a entender a mensagem
            sem esforço.
          </p>
          <p>
            Mesmo com um bom conteúdo, uma tipografia mal aplicada pode dificultar a
            leitura e causar abandono da página.
          </p>
        </section>

        <section>
          <h2>Hierarquia visual</h2>
          <p>
            A hierarquia visual orienta o leitor sobre o que é mais importante em uma
            página. Títulos devem chamar mais atenção, enquanto parágrafos devem ser
            confortáveis para leitura contínua.
          </p>
          <p>
            Em HTML, a hierarquia começa com o uso correto das tags de título, como
            <strong>&lt;h1&gt;</strong> e <strong>&lt;h2&gt;</strong>, e é reforçada
            visualmente com CSS.
          </p>
        </section>

        <section>
          <h2>Boas práticas para textos</h2>
          <p>
            Textos longos devem ter espaçamento adequado entre linhas e parágrafos.
            Isso cria pausas naturais e melhora a experiência de leitura.
          </p>
          <p>
            Além disso, limitar a largura do texto evita linhas muito longas, tornando
            a leitura mais fluida e confortável.
          </p>
        </section>
      </main>
  </body>
</html>

```

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS:**

```css
body {
    font-family: Arial, Helvetica, sans-serif;
    line-height: 1.6;
}

h1 {
    font-size: 2rem;
    margin-bottom: 0.5rem;
}

h2 {
     font-size: 1.4rem;
}

p {
	margin-bottom: 1rem;
}
```

<img src="https://i.imgur.com/3HPZ0By.png" title="source: imgur.com" width="3%"/>**Resultado:**

<div align="center"><img src="https://i.imgur.com/hWP5v12.png" title="source: imgur.com"/></div>

Observe, no exemplo acima, que a estrutura segue **boas práticas de semântica, hierarquia e legibilidade**:

- Utiliza **apenas um `<h1>`**, representando corretamente o tema principal da página
- Os títulos `<h2>` organizam seções de mesmo nível, criando uma hierarquia clara e previsível
- Os parágrafos (`<p>`) são usados exclusivamente para **texto corrido**, evitando uso indevido para títulos ou espaçamentos
- O uso de `line-height` com valor numérico garante **conforto visual e consistência tipográfica**
- A estrutura HTML mantém sentido e organização **mesmo sem CSS**, reforçando o uso correto da semântica

Esse modelo funciona como uma **base sólida e reutilizável** para páginas institucionais, artigos, blogs e conteúdos educacionais, pois respeita desde o início os princípios de hierarquia visual, semântica HTML e experiência de leitura.
