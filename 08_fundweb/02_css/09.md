<h1>Estilização de Listas</h1>



A propriedade **`list-style`** é utilizada no CSS para controlar a **aparência visual dos marcadores** das listas HTML, tanto **ordenadas (`<ol>`)** quanto **não ordenadas (`<ul>`)**. Ela permite definir o **tipo do marcador** e a sua **posição**.

Esse recurso é muito comum em menus, listas de funcionalidades, passos de um processo e conteúdos informativos.

`list-style` é uma **propriedade abreviada (shorthand)** que reúne três configurações:

- **`list-style-type`** → tipo do marcador
- **`list-style-position`** → posição do marcador
- **`list-style-image`** → imagem personalizada como marcador

<br />

> [!IMPORTANT]
>
> A propriedade `list-style-image` permite usar imagens como marcadores, mas seu uso é menos comum. Em layouts modernos, é mais recomendado utilizar pseudo-elementos (`::before`) para maior controle visual.

<br />

Essas propriedades podem ser usadas juntas ou separadamente, dependendo da necessidade.

<br />

<h2>1. Listas não ordenadas</h2>



Listas não ordenadas são usadas quando **a ordem dos itens não importa**, como listas de recursos ou tópicos.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
ul {
  list-style-type: disc;
}
```

**Valores comuns:**

- **`disc`** → padrão (bolinha preenchida)
- **`circle`** → círculo vazado
- **`square`** → quadrado
- **`none`** → remove o marcador

Veja na imagem abaixo os marcadores utilizados em listas não ordenadas:

<div align="center"><img src="https://i.imgur.com/5vYVXmD.png" title="source: imgur.com"/></div>

Note que a opção `none` oculta o marcador.

<br />

<h2>2. Listas ordenadas</h2>



Listas ordenadas são usadas quando **a sequência dos itens é importante**, como etapas, rankings ou instruções.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
ol {
  list-style-type: decimal;
}
```

**Valores comuns:**

- **`decimal`** → 1, 2, 3…
- **`decimal-leading-zero`** → 01, 02, 03…
- **`lower-alpha`** → a, b, c…
- **`upper-alpha`** → A, B, C…
- **`lower-roman`** → i, ii, iii…
- **`upper-roman`** → I, II, III…

Veja na imagem abaixo os marcadores utilizados em listas ordenadas:

<div align="center"><img src="https://i.imgur.com/0lO9Whf.png" title="source: imgur.com"/></div>

<h2>3. Posição do marcador</h2>



A propriedade `list-style-position` define **onde o marcador aparece em relação ao conteúdo** do item.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
ul {
  list-style-position: outside;
}
```

**Valores disponíveis:**

- **`outside`** → marcador fica fora do bloco do texto (padrão)
- **`inside`** → marcador faz parte do fluxo do texto

Veja na imagem abaixo a diferença entre `outside` e `inside`:

<div align="center"><img src="https://i.imgur.com/0lO9Whf.png" title="source: imgur.com"/></div>

<br />

<h2>4. A propriedade list-style</h2>



Além de configurar separadamente propriedades como `list-style-type` e `list-style-position`, o CSS permite utilizar a propriedade **shorthand** `list-style`, que reúne essas definições em uma única linha. Essa abordagem torna o código mais limpo, legível e fácil de manter.

<img src="https://i.imgur.com/DSBasbH.png" title="source: imgur.com" width="4%"/>**CSS - Exemplo:**

```css
ol {
  list-style: upper-roman outside;
}
```

Neste caso:

- `upper-roman` define a numeração em algarismos romanos maiúsculos.
- `outside` posiciona o marcador fora do conteúdo do item.

<br />

<h2>5. Removendo marcadores</h2>



Em menus de navegação, é uma prática comum remover completamente os marcadores e o espaçamento padrão aplicado pelos navegadores. Isso permite maior controle visual e facilita a criação de layouts personalizados.

```css
ul {
  list-style: none;
  padding: 0;
  margin: 0;
}
```

Essa configuração remove:

- Os marcadores da lista
- O espaçamento padrão (`padding` e `margin`) aplicado pelo navegador

Essa técnica é amplamente utilizada na construção de **menus horizontais, barras de navegação e listas estilizadas**, servindo como ponto de partida para layouts mais elaborados com Flexbox ou Grid.

<br />

## **Boas práticas**

- Use **listas ordenadas** apenas quando a ordem tiver significado
- Evite remover marcadores sem oferecer outro indicativo visual
- Prefira simplicidade para melhorar a leitura
- Em menus, combine `list-style: none` com flexbox ou grid
- Para maior personalização visual, considere pseudo-elementos

A propriedade `list-style` é essencial para adaptar listas ao design da interface, permitindo desde pequenos ajustes visuais até a criação de menus e componentes personalizados. Compreender suas variações ajuda a manter **clareza, semântica e organização visual** no HTML.
