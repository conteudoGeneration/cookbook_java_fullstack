<h1>Tipografia (Fontes)</h1>



A **tipografia** está presente em praticamente tudo o que consumimos visualmente: livros, sites, aplicativos, placas, sinais de trânsito, embalagens e interfaces digitais. Ela se refere ao **estilo, à aparência e à organização do texto**, sendo também considerada a **arte de trabalhar com letras** para tornar a comunicação clara, legível e visualmente agradável.

No contexto do **design digital e do desenvolvimento Frontend**, a tipografia vai muito além da escolha de uma fonte bonita. Ela influencia diretamente a **experiência do usuário**, a **leitura**, a **acessibilidade** e a **percepção da marca**. Assim como as cores, a tipografia tem forte impacto visual e deve ser escolhida com critério.

Ao criar uma interface, você está constantemente tomando decisões tipográficas: tamanho da fonte, espaçamento, peso, alinhamento e hierarquia entre títulos e textos. Por isso, compreender os fundamentos da tipografia é essencial para construir projetos mais profissionais e eficientes.

<h3>2.1. Boas práticas para escolha de fontes em projetos web</h3>



Ao definir a tipografia de um site ou aplicação, considere as seguintes recomendações:

- **Priorize legibilidade**: o texto deve ser fácil de ler em diferentes tamanhos de tela.
- **Use poucas fontes**: normalmente, uma fonte para títulos e outra para textos é suficiente.
- **Crie hierarquia visual**: diferencie títulos, subtítulos e parágrafos com tamanho, peso e espaçamento.
- **Considere o contexto**: o estilo da fonte deve estar alinhado com o público e o objetivo do projeto.
- **Teste combinações**: experimentar diferentes fontes ajuda a encontrar a melhor solução.
- **Pense em acessibilidade**: fontes muito finas ou decorativas podem prejudicar a leitura.
- **Utilize fontes web seguras ou serviços confiáveis**, como Google Fonts.

Não tenha receio de testar, ajustar e revisar suas escolhas: a tipografia é um dos elementos que mais evoluem conforme o projeto amadurece.

<h2>3. Fontes no CSS</h2>



Nos primeiros anos da web, o número de fontes disponíveis para uso em páginas era bastante limitado, pois dependia exclusivamente das fontes instaladas no sistema operacional do usuário. Felizmente, esse cenário evoluiu significativamente. Hoje, é possível utilizar praticamente **qualquer tipo de fonte** em projetos web, graças aos avanços do CSS e à popularização de bibliotecas de fontes online.

O CSS oferece um conjunto de propriedades específicas para o controle tipográfico, permitindo definir desde a família da fonte até seu tamanho, peso e estilo. As principais propriedades relacionadas a fontes são:

- **`font-family`** — define a família tipográfica
- **`font-size`** — controla o tamanho do texto
- **`font-weight`** — define o peso da fonte (normal, bold, etc.)
- **`font-style`** — define variações como itálico ou normal

Essas propriedades trabalham em conjunto para criar textos legíveis, organizados e visualmente coerentes com o layout da interface, como veremos mais adiante.

<br />

<h3>3.1. Propriedade `font-family`</h3>



A propriedade **`font-family`** define a *família de fontes* que será utilizada por um elemento. O termo “família” é usado porque uma fonte não é composta por um único arquivo, mas por um **conjunto de variações**, como:

- Regular
- Bold (negrito)
- Italic (itálico)
- Bold Italic

Quando você aplica, por exemplo, um texto em itálico com CSS, o navegador tenta utilizar a **subfonte correspondente** dentro da mesma família tipográfica, garantindo consistência visual e melhor renderização.

O CSS permite trabalhar tanto com **fontes padrão do sistema** quanto com **fontes externas**, importadas de serviços como o **Google Fonts**, ampliando consideravelmente as possibilidades de design.

<br />

<h3>3.2. Fontes Padrão do CSS (Web Safe Fonts)</h3>



As **fontes padrão** são aquelas que estão disponíveis na maioria dos sistemas operacionais, independentemente da plataforma. Elas são conhecidas como **Web Safe Fonts**, pois oferecem maior garantia de compatibilidade entre diferentes dispositivos e navegadores.

**Exemplo:**

```css
body {
  font-family: Helvetica;
}
```

Entretanto, é uma boa prática definir **fontes alternativas (fallbacks)**. Caso a fonte principal não esteja disponível — seja por ausência no sistema ou falha no carregamento — o navegador utilizará a próxima opção da lista:

```css
body {
  font-family: Helvetica, Arial;
}
```

Nesse caso, se o sistema não encontrar a Helvetica, ele utilizará a Arial.

<br />

<h4>3.2.1. Classificação das fontes padrão</h4>



As Web Safe Fonts são tradicionalmente classificadas em três grandes categorias:

| Serif           | Sans-serif  | Monospace      |
| --------------- | ----------- | -------------- |
| Georgia         | Arial       | Courier        |
| Times New Roman | Arial Black | Courier New    |
| Times           | Helvetica   | Lucida Console |
|                 | Verdana     | Monaco         |
|                 | Tahoma      |                |
|                 | Trebuchet   |                |

- **Serif**: possuem pequenos traços decorativos nas extremidades das letras.
- **Sans-serif**: não possuem serifas, sendo mais limpas e indicadas para telas.
- **Monospace**: cada caractere ocupa o mesmo espaço horizontal, ideais para código.

<br />

<h4>3.2.2. Fontes genéricas no CSS</h4>



Além de nomes específicos, o CSS permite utilizar **famílias genéricas**, que funcionam como uma última alternativa (*fallback*), garantindo que o navegador escolha uma fonte compatível com o estilo desejado:

- `sans-serif` — fontes sem serifas
- `serif` — fontes com serifas
- `monospace` — fontes monoespaçadas
- `cursive` — fontes manuscritas
- `fantasy` — fontes decorativas

Esses nomes genéricos são normalmente colocados **no final da declaração** de `font-family`:

```css
body {
  font-family: Helvetica, Arial, sans-serif;
}
```

Dessa forma, se nenhuma das fontes especificadas estiver disponível, o navegador selecionará automaticamente uma fonte **sans-serif padrão do sistema**, mantendo a legibilidade e o estilo geral da página.

Esse cuidado com a definição de fontes e fallbacks é essencial para criar interfaces **consistentes, acessíveis e confiáveis**, independentemente do dispositivo ou sistema operacional do usuário.

<h2>4. Fontes do Google</h2>



O **Google Fonts** é uma biblioteca gratuita mantida pelo Google que reúne **centenas de famílias tipográficas de código aberto**, licenciadas para uso livre em projetos pessoais e comerciais. A plataforma oferece uma interface web intuitiva para explorar as fontes e uma **API** que permite utilizá-las facilmente em projetos **HTML, CSS e Android**.

Uma das grandes vantagens do Google Fonts é a simplicidade de uso: não é necessário baixar arquivos manualmente. Basta incluir o código gerado no site e aplicar a fonte no CSS, como veremos a seguir:

<div align="center"><img src="https://i.imgur.com/Z5SwAM2.png" title="source: imgur.com" /></div>

9. O código gerado para a fonte **Poppins**, usada no exemplo acima, você confere abaixo:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Poppins:ital,wght@0,100;0,200;0,300;0,400;0,500;0,600;0,700;0,800;0,900;1,100;1,200;1,300;1,400;1,500;1,600;1,700;1,800;1,900&display=swap" rel="stylesheet">
```

10. Insira o código acima na seção **`<head>`** das páginas HTML do seu projeto, , antes do fechamento da tag.

```html
<!DOCTYPE html>
<html lang="pt-br">
<head>
    
    <!-- Código copiado do Google Fonts -->
    
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link href="https://fonts.googleapis.com/css2?family=Poppins:ital,wght@0,100;0,200;0,300;0,400;0,500;0,600;0,700;0,800;0,900;1,100;1,200;1,300;1,400;1,500;1,600;1,700;1,800;1,900&display=swap" rel="stylesheet">
    
    <!-- Fim do Código -->
    
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="icon" href="./assets/img/favicon.svg" />
    <title>Universe</title>
</head>
<body>
    
</body>
</html>
```

11. Após importar a fonte, basta utilizá-la no CSS com a propriedade `font-family`, como vemos no exemplo abaixo:

```css
font-family: 'Poppins', sans-serif;
```

O nome da fonte deve ser exatamente o mesmo indicado pelo Google Fonts (Poppins), seguido de uma fonte genérica como *fallback*.

<br />

<h3>4.2. Boas práticas ao usar Google Fonts</h3>



- Evite carregar muitas fontes no mesmo projeto
- O ideal é utilizar **no máximo duas famílias tipográficas** (por exemplo: uma para títulos e outra para textos)
- Prefira importar apenas os **pesos realmente necessários**. Observe que na importação padrão, todos os pesos foram importados (100 - 900)
- Sempre inclua uma **fonte genérica de fallback**
- Lembre-se de que cada fonte importada impacta no **tempo de carregamento da página**

Seguindo essas orientações, você garante um site visualmente consistente, profissional e com bom desempenho.
