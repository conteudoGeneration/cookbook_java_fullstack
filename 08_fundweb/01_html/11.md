<h1>Acessibilidade com ARIA Attributes</h1>



Os atributos **ARIA (Accessible Rich Internet Applications)** são uma tecnologia do HTML5 que torna as aplicações web mais acessíveis para pessoas com deficiências. 

Eles fornecem informações semânticas adicionais aos leitores de tela e tecnologias assistivas, permitindo que usuários com limitações visuais, auditivas ou motoras compreendam e interajam com o conteúdo. O ARIA não altera a aparência visual, mas enriquece a estrutura para acessibilidade universal.

**Sintaxe:**

```html
aria-nome-do-atributo="valor"
```

Os atributos ARIA seguem uma sintaxe simples:

- **aria:** indica que se trata de um atributo do tipo aria
- **nome_do_atributo:** nome do atributo
- **valor:** valor do atributo

Os atributos ARIA são aplicados em tags de abertura. Eles são divididos em categorias como papéis (roles), propriedades de estado (states) e propriedades (properties), ajudando a descrever elementos dinâmicos como menus expansíveis ou diálogos modais.

<img src="https://i.imgur.com/O5NjoiA.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<div role="button" aria-label="Fechar menu" tabindex="0">X</div>
```

<h2>1. Categorias de Atributos ARIA</h2>



Os atributos ARIA são organizados em grupos para diferentes finalidades, facilitando a implementação gradual.

| Atributo ARIA        | Função                                                 | Exemplo simples            | Uso comum                |
| -------------------- | ------------------------------------------------------ | -------------------------- | ------------------------ |
| **aria-label**       | Define um rótulo acessível quando não há texto visível | `aria-label="Buscar"`      | Botões e ícones          |
| **aria-labelledby**  | Usa outro elemento como rótulo                         | `aria-labelledby="titulo"` | Inputs e seções          |
| **aria-describedby** | Associa um texto de ajuda ou descrição                 | `aria-describedby="ajuda"` | Formulários              |
| **aria-hidden**      | Oculta o elemento das tecnologias assistivas           | `aria-hidden="true"`       | Ícones decorativos       |
| **aria-required**    | Indica que o campo é obrigatório                       | `aria-required="true"`     | Inputs                   |
| **aria-invalid**     | Indica valor inválido                                  | `aria-invalid="true"`      | Validação de formulário  |
| **aria-expanded**    | Indica estado aberto ou fechado                        | `aria-expanded="false"`    | Menus e acordeões        |
| **aria-checked**     | Indica estado marcado                                  | `aria-checked="true"`      | Checkboxes customizados  |
| **aria-disabled**    | Indica elemento desabilitado                           | `aria-disabled="true"`     | Botões e inputs          |
| **aria-live**        | Define região com atualizações dinâmicas               | `aria-live="polite"`       | Mensagens dinâmicas      |
| **role**             | Define a função semântica do elemento                  | `role="alert"`             | Componentes customizados |

A tabela apresenta apenas os **atributos mais usados no dia a dia**, com exemplos curtos e fáceis de relacionar com situações reais, mantendo uma abordagem clara e introdutória.

<br />

<img src="https://i.imgur.com/O5NjoiA.png" title="source: imgur.com" width="5%"/>**Exemplo:**

```html
<form>
    <label for="email">E-mail</label>

    <input 
        type="email"
        id="email"
        aria-label="Campo de e-mail"
        aria-describedby="emailHelp"
        aria-required="true"
    >

    <p id="emailHelp">
        Informe um e-mail válido.
    </p>

    <div role="alert">
        Campo obrigatório.
    </div>
</form>
```

### Relação entre atributos e o exemplo

| Atributo             | Onde aparece no exemplo | Finalidade prática        |
| -------------------- | ----------------------- | ------------------------- |
| aria-label           | `<input>`               | Nome acessível do campo   |
| aria-describedby     | `<input>` → `emailHelp` | Texto de ajuda ao usuário |
| aria-required="true" | `<input>`               | Indica campo obrigatório  |
| role="alert"         | `<div>`                 | Anúncio imediato de erro  |

O exemplo acima reflete um **cenário real do dia a dia**, comum em formulários.

<br />

<h2>2. Boas Práticas com ARIA</h2>



Sempre priorize HTML semântico nativo (como `<button>` em vez de `div role="button"`), usando ARIA apenas quando necessário para elementos personalizados via JavaScript. Teste com ferramentas como o validador ARIA do W3C e leitores de tela como NVDA ou VoiceOver.

- Evite ARIA em elementos que já possuem semântica nativa, como `<nav>` ou `<main>`.
- Use `aria-live="polite"` para atualizações dinâmicas, como notificações em chats.

<br />

> [!WARNING]
>
> Não utilize os atributos ARIA indiscriminadamente. Ele corrige HTML mal estruturado, mas o ideal é sempre usar as tags semânticas corretas para a acessibilidade nativa.
