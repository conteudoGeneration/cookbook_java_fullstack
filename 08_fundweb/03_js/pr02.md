<h1>Projeto 01 - Portfólio - Seção Projects - Consumo API - GitHub</h1>



Após consumir a API do GitHub para obter os **dados do perfil do usuário** exibidos na seção **About**, podemos utilizar o JavaScript para **carregar dinamicamente os últimos projetos** na seção **Projects**.

Na prática, iremos realizar uma **Requisição HTTP** para a **API do GitHub**, consultando os **últimos repositórios públicos** do usuário, e a partir da resposta:

- Geraremos **cards para cada projeto** que serão adicionados no carrossel;
- Exibiremos informações como:
  - Nome do repositório (nome do projeto);
  - Descrição do projeto;
  - Link direto para o repositório;
  - Link direto para o deploy (se disponível);
  - Linguagem utilizada;
  - Tags do projeto (Topics)

Esses cards serão inseridos automaticamente no **carrossel da seção Projects**, permitindo que os projetos mais recentes sejam exibidos de forma dinâmica e sempre atualizada.

Dessa forma, os dados carregados diretamente da API garantem que a seção **Projects** esteja sempre **atualizada**, sem a necessidade de alterar manualmente o código HTML.

<br />

> [!WARNING]
>
> **Pontos de Atenção**
>
> - A API do GitHub **só retorna informações de repositórios públicos**.
> - Para que os cards do carrossel na seção **Projects** sejam gerados corretamente, **é fundamental** que a **seção About** de cada repositório esteja **devidamente preenchida com as informações do projeto (como descrição, linguagem, link do deploy e os topics)**. Caso esses dados não estejam preenchidos, os cards podem aparecer incompletos ou com informações padrões que não refletem o seu projeto. Veja um exemplo abaixo:

**Linha 81:** Define a função assíncrona `getProjectsGithub()`. Por ser assíncrona, ela permite utilizar `await` para realizar requisições à API do GitHub de forma sequencial, sem bloquear a execução do restante do script.

<br />

> [!TIP]
>
> A URL utilizada para buscar os dados dos Repositórios públicos  no GitHub segue o padrão:
>
> ```js
> https://api.github.com/users/nome_usuario/repos?sort=updated&per_page=6
> ```
>
> Onde:
>
> - `nome_usuario` é o seu usuário no GitHub. **Não esqueça de alterar!**
>
> Esse endpoint foi obtido na **documentação oficial da API do GitHub** e retorna os repositórios públicos de um usuário, permitindo ordenar os resultados (`sort=updated`) e limitar a quantidade de itens retornados (`per_page=6`).
>
> No código, esses dados são utilizados para criar dinamicamente os cards de projetos, exibindo informações como nome, descrição, linguagem, tags e links para GitHub e deploy.

**Linha 86:** Converte a resposta da requisição para o formato JSON, permitindo acessar os dados dos repositórios como objetos JavaScript.

**Linha 88:** Limpa o conteúdo atual do container `swiperWrapper`, garantindo que qualquer card de projeto exibido anteriormente seja removido antes de inserir os novos.

**Linha 91 a 106:** Define o objeto `linguagens`, que associa cada linguagem de programação a um ícone correspondente. Esse mapeamento será usado para exibir o ícone correto no card de cada repositório.

**Linhas 108 a 173:** Inicia um loop `forEach` que percorre cada repositório retornado pela API, permitindo processar os dados de cada projeto individualmente.

**Linha 111:** Define o objeto `linguagem`, que recebe a linguagem principal do repositório. Caso o repositório não tenha uma linguagem definida, utiliza `'GitHub'` como padrão.

**Linha 114:** Obtém a configuração do logo correspondente à linguagem principal do projeto, garantindo que sempre exista uma imagem válida. Caso o repositório não tenha uma linguagem padrão definida, utilizaremos o logo do `'GitHub'` como padrão.

**Linha 117:** Monta a URL que apontará para a imagem do logo que será exibido no card do projeto. Os logos estão disponíveis na pasta `assets/icons/languages`.

**Linhas 120 a 123:** Formata o nome do repositório, substituindo hifens (-) e underlines (_) por espaços em branco, removendo caracteres especiais da string e convertendo o texto para letras maiúsculas. Desta forma, o título do projeto ficará mais legível no card.

**Linhas 126 a 128:** Implementa uma função `truncar`, responsável por limitar o tamanho da descrição do repositório. Quando o texto ultrapassa 100 caracteres, a função retorna os 97 primeiros caracteres seguidos de reticências (...), indicando a continuidade do conteúdo. Caso a descrição possua até 100 caracteres, o texto original é retornado sem alterações.

**Linhas 131 a 133:** Define a descrição do projeto. Se houver uma descrição, executa a função `truncar`, criada a anteriormente, para ajustar o tamanho do texto. Caso não haja descrição, exibe o texto padrão: `'Projeto desenvolvido no GitHub'`.

**Linhas 136 a 138:** Responsável por gerar as tags do projeto. Quando o repositório possui tópicos (`topics`), são exibidos no máximo três. Caso não existam tópicos cadastrados, é apresentada uma única tag com a linguagem principal do repositório.

Para limitar a quantidade de tópicos, utiliza-se o método `slice`, que seleciona apenas os três primeiros itens do array. Em seguida, o método `map` é aplicado para percorrer esses itens e gerar dinamicamente as respectivas tags.

**Linhas 141 a 143:** Cria a constante `botaoDeploy` que verificará se o repositório possui o link do *deploy* (endereço da home page). Se existir, ele gera o botão com o link de acesso ao *deploy*.

**Linhas 146 a 153:** Cria os botões de ação do card (Link do GitHub e Link do Deploy, se disponível). 

**Linhas 155 a 172:** Monta o HTML do card do projeto e adiciona ao container `swiperWrapper`. Inclui:

- Imagem do logo da linguagem, com fallback para o logo do GitHub caso haja erro;
- Nome formatado do repositório;
- Descrição do projeto;
- Tags do projeto (Topics);
- Botões de ação (GitHub + Deploy).

**Linha 176:** Executa a função `iniciarSwiper()` após a inserção dos cards na DOM, garantindo que o carrossel seja inicializado corretamente e que os cards de projetos sejam exibidos dentro do componente de *slider* na página.
